# AI 对机器友好的设计范式规范

## 1. 核心目的
构建"机器优先、人类可用"的系统架构，确保自动化、可测试性与可组合性。

## 2. 模块规范

### 2.1 第一层：命令行工具 (CLI)
- **定位**：机器与人类的共同入口，系统核心基础。
- **输出模式**：
  - 默认：人类可读格式
  - `--json` / `--yaml`：机器可读格式
- **命令结构**：
  - 使用子命令划分功能层次
  - 参数随子命令带入，层级清晰
- **要求**：
  - 无状态（Stateless）
  - 无交互（Non-interactive）
  - 明确退出码

### 2.2 第二层：结构化 UI 配置方案
- **定位**：基于 CLI 能力的标准化封装。
- **要求**：
  - 配置即代码，支持版本控制与复用。
  - 作为连接 CLI 与人类 UI 的唯一中间层。

### 2.3 第三层：人类友好 UI
- **定位**：顶层可视化交互界面。
- **要求**：
  - 仅调用结构化配置方案。
  - 不包含核心业务逻辑。
  - 故障时可降级至 CLI 使用。
